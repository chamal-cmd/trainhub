# Aircall Call Reports Dashboard — Project Instructions

## What this project is

A live call reporting dashboard for Aircall that shows real-time call data and uses Claude AI to analyze individual calls. It runs as a local web app on your computer, accessible at **http://localhost:4001**.

---

## How to access the dashboard

Open your browser and go to: **http://localhost:4001**

The server runs automatically in the background every time you log into Windows (set up via Task Scheduler + PM2). You do not need to start it manually.

If the dashboard is not loading, open a terminal and run:
```
pm2 restart call-reports
```

## How the data flows

1. Browser loads `http://localhost:4001` → served by `server.js`
2. Browser calls `/api/calls?from=TIMESTAMP&to=TIMESTAMP`
3. `server.js` calls `https://api.aircall.io/v1/calls` with your credentials
4. Aircall returns real call records
5. `server.js` maps each call to the dashboard schema and sends it back
6. `app.js` renders KPI cards, charts, filters, and the calls table
7. When you click a call row → detail panel opens
8. Clicking "Re-analyze with Claude" → `app.js` calls the Anthropic API directly from the browser and streams the AI response

## Anthropic (Claude) API key

Entered via the **⚙ Settings** icon in the dashboard header. Stored in the browser's memory only — you need to re-enter it each time the page is refreshed.

The key is used for the **"Re-analyze with Claude"** button in the call detail panel. It streams a live AI analysis of the selected call.

## Dashboard features

| Feature | How it works |
|---|---|
| **Date range** | Top bar buttons (Today / This Week / This Month) — refetches from Aircall |
| **KPI cards** | Total Calls, Avg Duration, Positive Sentiment %, Follow-ups Needed |
| **Sentiment chart** | Donut chart — all calls default to "neutral" until re-analyzed with Claude |
| **Outcomes chart** | Bar chart — Resolved / Unresolved / Follow-up Required / Sale Made |
| **Search** | Searches phone number, agent name, topics, and tags |
| **Agent filter** | Dropdown populated from actual agents in the loaded calls |
| **Sentiment filter** | Toggle buttons — can select multiple |
| **Outcome filter** | Dropdown |
| **Sort** | Click any column header to sort ascending/descending |
| **Expand row** | Click the ↓ arrow at the end of any row to see topics and action items inline |
| **Detail panel** | Click any row to open a dark sidebar with full call details |
| **Re-analyze with Claude** | Streams AI summary, sentiment update, and action items for any call |
| **Export CSV** | Downloads the currently filtered calls as a CSV file |

## Call data schema

Each call loaded from Aircall has these fields:

```js
{
  id:                 string,   // Aircall call ID
  agent:              string,   // Agent name from Aircall
  phone:              string,   // Caller phone number
  direction:          string,   // "inbound" or "outbound"
  duration:           number,   // Duration in seconds
  timestamp:          Date,     // Call start time
  status:             string,   // "done", "missed", etc.
  tags:               string[], // Aircall tags on the call
  sentiment:          string,   // "positive" | "neutral" | "negative" (default: neutral)
  sentiment_score:    number,   // 0.0 to 1.0 (default: 0.5)
  call_outcome:       string,   // "resolved" | "unresolved" | "follow-up-required" | "sale-made"
  key_topics:         string[], // AI-generated topics (empty until re-analyzed)
  action_items:       string[], // Notes/comments from Aircall, or AI-generated
  follow_up_required: boolean,  // true if status is "missed"
  summary:            string,   // AI summary (placeholder until re-analyzed)
  contact_name:       string,   // Contact name from Aircall
  recording:          string,   // Recording URL if available
}
```

Sentiment, key_topics, and action_items are **not stored in Aircall** — they are generated by Claude when you click "Re-analyze with Claude" on a call. Currently each session starts with defaults.

## How to make changes

### Change a visual style or layout
Edit `public/index.html` (for structure) or `public/app.css` (for custom animations/styles). Most styling uses Tailwind CSS classes directly in the HTML.

### Change dashboard logic (filters, sorting, KPIs, etc.)
Edit `public/app.js`. Key functions:
- `loadCalls(range)` — fetches from `/api/calls`, called on load and refresh
- `applyFilters()` — filters + sorts `ALL_CALLS` into `filtered`, then re-renders everything
- `renderKPIs()` — updates the 4 KPI cards
- `renderCharts()` — redraws both Chart.js charts
- `renderTable()` — rebuilds the calls table HTML
- `openPanel(call)` — opens the detail sidebar
- `reanalyze(id)` — calls the Anthropic API with streaming

### Change how Aircall data is mapped
Edit the `mapCall()` function in `server.js`. This transforms a raw Aircall call object into the dashboard schema above.

### Change the Claude analysis prompt
Edit the `prompt` variable inside the `reanalyze()` function in `public/app.js` (around line 280).

### Add a new Aircall API field
1. Add it to `mapCall()` in `server.js`
2. Display it in `openPanel()` in `public/app.js`

### Restart the server after changes to server.js
```
pm2 restart call-reports
```
Changes to `public/` files (HTML, JS, CSS) take effect on the next browser refresh — no server restart needed.

## Troubleshooting

| Problem | Fix |
|---|---|
| Dashboard not loading | Run `pm2 restart call-reports` in a terminal |
| "Demo data" badge showing | Aircall credentials wrong — check `server.js` lines 21–22 |
| "Re-analyze" button does nothing | Enter Anthropic API key via ⚙ Settings icon |
| Port 4001 already in use | Run `pm2 kill` then `pm2 start server.js --name call-reports` |
| Check server logs | Run `pm2 logs call-reports` |
| See all PM2 processes | Run `pm2 list` |
